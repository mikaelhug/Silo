Silo

A native macOS launcher for your Windows Steam library — run games through Wine and Apple's Game Porting Toolkit, each in its own isolated runtime, with a real Steam client so Steamworks just works.

Built for the Mac

Your Windows Steam library, in a native launcher

Your real Steam library

Silo discovers the games installed in its Steam bottle and launches each one under GPTK — co-resident with a logged-in Steam client, so Steamworks and DRM titles run.

Apple GPTK graphics

D3DMetal maps DirectX 10/11/12 and DXGI straight to Metal; DirectX 9, 8, and OpenGL fall back to Wine's own renderer. No DXVK, no Homebrew.

A bottle per game

Add any Windows .exe — an installer or a portable build — and Silo provisions a dedicated, isolated Wine prefix for it. Relocate every bottle to an external drive anytime.

Self-contained & updating

Silo fetches its own Wine runtime (built only from CrossOver's FOSS source) and updates itself from GitHub Releases. No package manager, no external dependencies.

How a launch works

Single downloader, isolated runtimes

Steam is installed once into a single master bottle that only downloads games. Each game then launches in its own runtime — graphics backend and environment injected at launch — so one finicky title can't disturb the rest of your library.

1

Discover

Parse the bottle's appmanifest_*.acf for installed games, skipping Steam's redistributables.

2

Provision

Seed the game's Wine prefix with an idempotent wineboot.

3

Link graphics

Overlay Apple GPTK / D3DMetal into the runtime for DirectX → Metal.

4

Launch

Spawn the game detached, with WINEPREFIX pointed at its isolated bottle.

What runs, and how

Modern DirectX on Metal, the rest on Wine

DirectX 12 / 11 / 10 + DXGI GPTK · D3DMetal → Metal
DirectX 9 / 8 / DirectDraw Wine wined3d → OpenGL
OpenGL Wine → macOS OpenGL
Vulkan (native) not supported

Apple's Game Porting Toolkit accelerates modern DirectX; Wine's own stack covers the rest. The game drives the API — Silo picks the translation.

Get Silo

Install the current release

Silo runs on macOS 15 or newer on Apple Silicon, outside the App Sandbox so it can drive Wine. It is not yet notarized, so a manual install needs the quarantine step shown in the README.
